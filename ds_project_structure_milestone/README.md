# Data Science Project Folder Structure Milestone

## Objective

Create a clean, reusable, and collaboration-ready folder structure for Data Science work.

## Folder Layout

```text
ds_project_structure_milestone/
├── data/
│   ├── raw/                # Original source data (never edited)
│   └── processed/          # Cleaned/transformed datasets
├── notebooks/              # Jupyter notebooks for exploration and analysis
├── scripts/                # Reusable Python scripts (ETL, utilities, training)
├── outputs/
│   ├── figures/            # Charts/plots/images generated by analysis
│   └── reports/            # Final exports, summaries, and presentation artifacts
└── README.md               # Project structure explanation
```

## Why This Structure Works

- Keeps **code, data, and outputs** clearly separated.
- Protects `data/raw` from accidental overwrite.
- Makes file paths predictable across environments.
- Helps teammates quickly understand where files belong.
- Scales cleanly as project complexity grows.

## Collaboration Rules

- Use lowercase folder names for consistency.
- Save incoming datasets only in `data/raw`.
- Write transformed files to `data/processed`.
- Keep exploratory work in `notebooks` and reusable logic in `scripts`.
- Save generated artifacts only in `outputs`.

## 2-Minute Video Walkthrough Checklist

1. Show the root folder and subfolders.
2. Explain each folder's responsibility.
3. Highlight separation of raw data, notebooks/scripts, and outputs.
4. Explain how this prevents broken paths and accidental data corruption.
5. Conclude with how the structure supports collaboration and reuse.

## Milestone Scope

This milestone focuses on **organization only**. No dataset, analysis, or model output is required.
